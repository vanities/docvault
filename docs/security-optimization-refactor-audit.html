P0 Critical

Arbitrary file read and Zip Slip via download year traversal

SecurityBackendFile disclosure

Impact: POST /api/download/zip and POST /api/download/cpa-package join caller-controlled year beneath the entity path without containment checks. A traversal value can scan and zip files outside the entity directory. ZIP entry names can also contain ../, creating a Zip Slip hazard for recipients.

Fix

  • Validate year as a known year/directory selector, preferably /^(19|20)\d{2}$/ for tax exports.
  • Use resolveUnder(entityPath, year) before fs.access and scanDirectory.
  • When reading each scanned file, resolve it with resolveUnder(entityPath, file.path) and reject null.
  • Normalize ZIP entry names and reject absolute paths or any .. path segment.
  • Add regression tests with synthetic data for ../../outside and Zip Slip entry names.

P1 High

Authentication fails open by default while destructive/admin APIs remain reachable

SecurityAuthAdmin surface

Impact: When DOCVAULT_USERNAME and DOCVAULT_PASSWORD are unset, all API routes are treated as authenticated. That includes file mutation, settings mutation, backup/restore, external sources, and custom jobs that can write executable scripts and spawn child processes with inherited environment variables.

Fix

  • Fail closed in production/container mode. Require auth unless DOCVAULT_AUTH_DISABLED=true is explicitly set.
  • Bind to localhost by default; require explicit opt-in for 0.0.0.0.
  • Gate jobs, settings, backup/restore, file mutation, and external-source APIs behind an admin-only check.
  • Disable custom jobs by default unless an explicit DOCVAULT_CUSTOM_JOBS_ENABLED=true is configured.
  • Have Docker entrypoint validate required auth/secret configuration.

Same-origin document preview/open path can execute active uploaded content

SecurityFrontend + BackendStored active content

Impact: The frontend opens and embeds /api/file/... content as same-origin. The server serves extensions like HTML, JS, CSS, and SVG with active MIME types and no Content-Disposition, nosniff, or sandbox. A stored HTML/SVG file can execute on DocVault’s origin and call same-origin APIs. window.open(..., '_blank') also omits noopener.

Fix

  • Serve vault files as attachments by default; allow inline only for a strict preview allowlist such as PDF and safe raster images.
  • Add X-Content-Type-Options: nosniff.
  • For inline preview responses, add Content-Security-Policy: sandbox where compatible.
  • Use window.open(fileUrl, '_blank', 'noopener,noreferrer') and set opener = null defensively.
  • Sandbox the PDF iframe or route previews through a dedicated safe viewer endpoint.

P2 Medium

Symlink escape bypasses string-only path containment

SecurityFilesystem

Impact: resolveUnder() compares resolved strings, but file operations use fs.stat/fs.readFile, which follow symlinks. A symlink inside an entity can point outside DATA_DIR; serving, scanning, parsing, and zipping can follow it. The nearby comment says “resolving symlinks” but no realpath() is enforced. 

// server/data.ts:717-721
export function resolveUnder(baseDir: string, relPath: string): string | null {
  const base = path.resolve(baseDir);
  const target = path.resolve(baseDir, relPath);
  if (target !== base && !target.startsWith(`${base}${path.sep}`)) return null;
  return target;
}

Fix: Either reject symlinks during scans with lstat(), or compare realpath(target) against realpath(base) before following. Add tests with a temp symlink under an entity pointing outside the data dir.
